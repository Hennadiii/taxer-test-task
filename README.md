#  Playwright Certificate E2E Tests

Автоматизоване E2E тестування завантаження та верифікації цифрових сертифікатів на веб-платформі [js-qvfsmdwp.stackblitz.io](https://js-qvfsmdwp.stackblitz.io/).

---

##  Опис

Тести перевіряють повний цикл роботи з сертифікатами:

1. **Завантаження** — файл передається у dropzone через емуляцію drag & drop
2. **Верифікація списку** — перевіряється що в боковій панелі з'явився елемент з правильним ім'ям власника (Common Name)
3. **Верифікація деталей** — після кліку на сертифікат перевіряються всі поля таблиці: `SubjectCN`, `IssuerCN`, `ValidFrom`, `ValidTill`

Дані для перевірки (expected values) **динамічно витягуються** з самого файлу сертифіката перед початком тесту — жодних захардкоджених значень.

| Інструмент | Призначення |
|------------|-------------|

##  Встановлення та запуск

### 1. Встановити залежності

```bash
npm install
```

### 2. Встановити браузер Chromium

```bash
npx playwright install chromium
```

### 3. Додати сертифікати

Покласти файли `.cer` або `.crt` у папку:

```
tests/fixtures/
```

### 4. Запустити тести

```bash
# Headless режим
npm test

# З відкритим браузером
npm run test:headed

# Режим дебагу (крок за кроком)
npm run test:debug
```

Запустити один конкретний файл — використовуй унікальну частину імені (кирилиця та спецсимволи в `--grep` можуть не спрацювати):

```bash
# Латинські імена — повністю або частково
npx playwright test --grep "cert.cer"
npx playwright test --grep "privat"
npx playwright test --grep "idd"

# Кириличні імена — використовуй унікальний числовий суфікс
npx playwright test --grep "8101900"
npx playwright test --grep "8101916"
```

### 5. Переглянути HTML звіт

```bash
npm run report

# Якщо порт 9323 зайнятий — вказати інший
npx playwright show-report --port 9324
```

##  Як працює парсер сертифікатів

Файл `utils/cert-parser.ts` читає DER-закодований X.509 сертифікат та витягує:

| Поле | OID | Опис |
|------|-----|------|
| `commonName` | `2.5.4.3` | Ім'я власника сертифіката |
| `issuerName` | `2.5.4.3` | CN видавця (АЦСК) |
| `validFrom` | — | Дата початку дії |
| `validTo` | — | Дата закінчення дії |

Підтримувані формати:
- **DER** (бінарний) — основний формат українських АЦСК
- **PEM** (Base64 з header/footer) — автоматично конвертується в DER

Дати зберігаються в двох форматах:
- `validFrom` — повний: `2015-04-08 21:00:00 UTC`
- `validFromShort` — короткий: `2015-04-08`

##  Очікувана поведінка тестів

| Стан веб-застосунку | Результат тестів |
|---------------------|-----------------|
| UI відображає дані коректно | ✅ Тести проходять |
| UI відображає неправильні дані | ❌ Тести падають із чітким повідомленням що саме не збігається |

> Тести є **regression suite** — вони зафіксують момент коли застосунок буде виправлений і почне коректно відображати дані з сертифікатів.

##  Утиліти для діагностики

### Перевірити формат сертифікатів

```bash
npx ts-node scripts/diagnose-certs.ts
```

Виводить для кожного файлу: розмір, hex-заголовок, структуру ASN.1, значення полів дат.

### Дослідити структуру сторінки

```bash
npx ts-node scripts/debug-iframe.ts
```

Відкриває браузер і виводить всі `<iframe>` елементи, frames та кнопки на сторінці.
